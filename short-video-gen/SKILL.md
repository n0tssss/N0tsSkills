# 短视频生成

**质量在设计阶段决定，不在渲染后修补。**

## 执行铁律

1. **Stage 2 设计先于 Stage 3 编码** — 没画完设计稿不准写 HTML
2. **每个阶段产出独立文件** — 选题/设计/脚本各自存为 .md
3. **质量自查** — 每阶段结束时 AI 自己审一遍再进入下一阶段

## 快速启动

缺参数问用户，不缺直接进入 Stage 1。

---

## Stage 1: 选题 & 竞品

产出 `01-选题.md`。

必须包含：
- 一句话卖点
- 竞品对比（至少找 3 个同类内容，对比播放量/角度/优缺点）
- 差异化策略

**自查**：这个选题有人看吗？和别人有什么不同？

---

## Stage 2: 设计（最关键）

**没有通过设计自查，不准写 HTML。**

产出 `02-设计.md`，必须包含：

### 2.1 视觉方案总览
- 配色（底色/主色/辅色/强调色/文字色）
- 字体方案（标题/正文/强调的字号字重）
- 整体氛围描述（3 个形容词）

### 2.2 段落设计（歌词MV）/ 场景设计（其他类型）

**歌词MV 必须逐段落设计**：intro / verse / chorus / bridge / outro

每个段落明确：
```
段落：Verse 1 (0:18-0:60)
底色：深红黑 #1a0000
氛围：紧张、攻击性
文字颜色：白色主文字 + 红色关键词
特效：重音时文字抖动(scale 1→1.15→1, 0.1s)、关键词放大变红
排版：双行居中，上行36px副文，下行56px正文
背景：暗红orb + 细密网格
```

**非歌词MV 必须逐场景设计**：布局 + 素材 + 文字 + 动画

### 2.3 设计自查

打印并逐项确认：

```
🎨 设计自查
━━━━━━━━━━━━━━━━━━━━━━━━
□ 配色有层次感吗？（不只一个底色到底）    [Y/N]
□ 字体有大小对比吗？（最大≥72px）         [Y/N]
□ 每个段落/场景视觉有明显区别吗？          [Y/N]
□ 有动态特效设计吗？（不只淡入淡出）       [Y/N]
□ 关键帧/moment有特殊处理吗？             [Y/N]

全Y → 进入 Stage 3 / 否则 → 继续完善设计
```

---

## Stage 3: 脚本 & 素材

产出 `03-脚本.md`。

每个场景/段落标注：时间、素材文件、画面描述、文字内容、动画。

### 素材自查

```
📦 素材自查
━━━━━━━━━━━━━━━━━━━━━━━━
□ 每个场景都指定了具体素材文件？          [Y/N]
□ 素材和设计稿的段落/场景一一对应？        [Y/N]
□ 素材目录文件数 ≥ 场景数？               [Y/N]

全Y → 进入 Stage 4 / 否则 → 补充素材
```

---

## Stage 4: 实现

写 HTML + CSS + GSAP。严格按设计稿和脚本执行，不自由发挥。

渲染前打印：

```
🔧 渲染前自查
━━━━━━━━━━━━━━━━━━━━━━━━
□ HTML 实现了设计稿的每个段落/场景？       [Y/N]
□ 每个场景用了不同素材/背景？              [Y/N]
□ 特效按设计稿实现了？（抖动/变色/缩放等）  [Y/N]
□ 字体层级和设计稿一致？                   [Y/N]

全Y → 渲染 / 否则 → 修改 HTML
```

```bash
npm run check && npm run render
cp renders/*.mp4 ../rendered.mp4
```

## 工具

kimi-webbridge · HyperFrames · FFmpeg · 小米 MiMo TTS

## 环境

Node.js ≥ 22 · FFmpeg · kimi-webbridge

## HyperFrames 规范

`class="clip"`、`data-start/duration/track-index`、GSAP `paused:true` → `window.__timelines`、`tl.set()` hard kill、不用 `Math.random()`/`Date.now()`
