# 短视频生成

工业化创作流水线。7 阶段，4 道审核关卡。**审核不通过打回重做。**

参考标准：T/CECC 35-2024《商用短视频制作要求》、GY/T 405-2024《网络视听节目技术质量要求》、巨量星图内容制作规范。

## 工作目录

```
D:\视频生成\<project>/
  01-选题报告.md
  02-研究笔记.md
  03-素材清单.md
  04-视频脚本.md
  05-设计稿.md
  素材/          # ≥8 文件，≥4 不同内容图，含 logo
  index.html
  rendered.mp4
```

## 输出

默认 1920×1080。用户可指定。

## 参数

缺了用 `AskUserQuestion` 问（时长/视觉风格/比例），不缺直接开始。用户说"你定"就跳过。

## 流水线 + 审核关卡

### Stage 1: 选题策划 → `01-选题报告.md`

**目标**：基于竞品数据确定「为什么这个视频值得看」和「怎么做出差异化」。

#### 1.1 竞品调研（必须先做）

用 WebSearch 搜索同类视频，收集真实数据：

**搜索策略**：
```
# 搜 B 站
site:bilibili.com <主题关键词> 视频

# 搜抖音
site:douyin.com <主题关键词>

# 搜 YouTube  
site:youtube.com <topic keywords> video

# 搜公众号/知乎（看图文端有没有人写过）
site:mp.weixin.qq.com <主题关键词>
site:zhihu.com <主题关键词>
```

**竞品分析表**（找 3-5 个同类视频/内容）：

| # | 平台 | 标题 | 播放/阅读 | 互动(点赞/评论) | 时长 | 角度/切入点 | 做得好 | 做得差 |
|---|------|------|----------|---------------|------|-----------|--------|--------|
| 1 | B站 | XXX | 1.2万 | 356/42 | 3min | 教程向 | 步骤清晰 | 节奏太慢 |
| 2 | 抖音 | XXX | 5.6万 | 1200/89 | 45s | 炫技向 | 画面震撼 | 没讲清楚是什么 |
| 3 | ... | ... | ... | ... | ... | ... | ... | ... |

**关键数据提取**：
- 这个题材最高播放量是多少？（天花板在哪）
- 高播放视频的共同特点是什么？
- 低播放视频的通病是什么？
- 评论区观众在问什么？（没被满足的需求）
- 有没有明显的空白地带？（某角度没人做过）

#### 1.2 差异化策略

基于竞品分析，确定我们的切入点：

```markdown
## 竞品缺口 & 我们的机会

| 竞品没做好的 | 我们怎么做 |
|------------|----------|
| 节奏太慢，3分钟才进正题 | 前3秒直接钩子 |
| 只有教程向，没人做情怀向 | 用名场面+角色驱动 |
| 没有中文内容/中文内容质量差 | 中文配音+本土化叙事 |
| 画面全是录屏，缺乏设计感 | 高质量素材+视觉设计 |
```

#### 1.3 选题决策

```markdown
# 选题报告

## 一句话卖点
看完能获得什么？

## 目标受众 & 观看场景
谁看？什么场景下看？

## 核心钩子（前 3 秒）
基于竞品数据选钩子类型

## 差异化策略
和同类视频的具体区别

## 竞品数据摘要
- 同类最高播放：XXX
- 同类平均播放：XXX  
- 未被满足的需求：XXX
- 我们的空白地带：XXX

## 预估流量潜力
- 受众规模：大众 / 中等 / 垂直小众
- 传播驱动力：话题性 / 实用性 / 情绪共鸣 / 娱乐性
- 搜索价值：有人主动搜吗？搜索量预估
- 时效性：长期有效 / 短期热点 / 周期性
```

#### Gate 1: 选题审核

## 核心钩子（前 3 秒）

```

### Gate 1: 选题审核

对照以下每一项，**至少 3 项达到"强"才能通过**：

| 维度 | 弱 | 中 | 强 |
|------|-----|-----|-----|
| 钩子冲击力 | 平淡叙述 | 有点意思 | 必须停下来看 |
| 受众规模 | <1万人关心 | 1-10万人 | >10万人关心 |
| 传播潜力 | 看完就忘 | 可能点赞 | 想转发/分享 |
| 差异化 | 和别人一样 | 略有不同 | 角度独特 |
| 执行可行性 | 素材难找 | 可收集 | 素材充足 |

**判定**：
- ≥3 项「强」→ 通过
- <3 项「强」→ **打回 Stage 1**，重新选题或换角度

### Stage 2: 内容研究 → `02-研究笔记.md`

**kimi-webbridge 实地调研**（网站类必须做）：
- 逐个页面截图：首页、关于、产品、案例、联系
- 记录真实数据（年份、规模、用户数）
- 提取可引用的文案和 slogan
- 下载 logo

**开源项目**：读 README、下载官方截图/Logo、记录 Stars/下载量。

### Stage 3: 素材收集 → `03-素材清单.md`

**硬性门槛**（不达标不进 Stage 4）：
- [ ] 素材目录 ≥8 个文件
- [ ] ≥4 张不同内容的图（不同页面/角度/产品）
- [ ] 有 logo
- [ ] 至少 1 张 ≥1280px 宽的高清大图

---

### Gate 2: 素材审核

| 维度 | 检查项 | 不通过 |
|------|--------|--------|
| 数量 | ≥8 个文件 | 回 Stage 2/3 继续收集 |
| 多样性 | ≥4 张不同内容的图 | 同上 |
| 冲击力 | 有能全屏铺满震撼画面的大图 | 专门去找一张 |
| 品牌 | 有 logo | 截取或下载 |
| 真实性 | 有真实产品/界面截图（非纯文字排版） | 去截 |

---

### Stage 4: 脚本创作 → `04-视频脚本.md`

```markdown
| # | 时间 | 布局类型 | 素材 | 画面描述 | 画面文字 | 台词(TTS) | 动画 |
```

原则：
- 前 3 秒必须有钩子
- 每 5-8 秒换场景
- 每个场景用不同素材图
- 每屏文字 ≤15 字
- 结尾必须有记忆点
- 场景布局类型数 ≥ 总场景数 × 60%

---

### Gate 3: 内容审核（最关键的关卡）

**3.1 内容丰富度**

| 检查项 | 标准 |
|--------|------|
| 信息密度 | 每场景传达 1 个明确信息，不空洞不堆砌 |
| 视觉多样性 | 不同场景用不同素材图（不重复用图） |
| 场景类型 | 布局类型数 ≥ 场景数 × 60%（8 场景至少 5 种不同布局） |
| 数据支撑 | 至少 1 个场景有具体数字/年份/规模 |
| 素材利用率 | 素材目录里的图，至少 70% 在脚本中被用到 |

**3.2 精彩度**

| 检查项 | 标准 |
|--------|------|
| 钩子测试 | 看完前 3 秒的脚本描述，想继续看下去吗？ |
| 一句话总结 | 看完整个脚本能说出一句话总结吗？说不出来 → 主线不清 |
| 情绪曲线 | 有起伏吗？（开场强→展开→高潮→收尾）还是一直平？ |
| 记忆点 | 结尾有没有让人记住的东西？一个数字/一句话/一个画面 |

**3.3 合理性**

| 检查项 | 标准 |
|--------|------|
| 时长合理 | 信息量和时长匹配吗？太长会拖、太短说不清 |
| 受众匹配 | 目标受众能看懂吗？专业术语过多？过于浅显？ |
| 逻辑连贯 | 场景顺序合理吗？没有突兀的跳跃 |

**3.4 合规检查**

| 检查项 | 标准 |
|--------|------|
| 无绝对化用语 | 没有"最好""第一""唯一""全网最" |
| 无可考证承诺 | 没有"100%""纯天然""月入过万""三天见效" |
| 无违规引导 | 没有非官方二维码/微信号/外链 |
| 版权安全 | 素材来源合规（官方截图/自己截的/CC0/已授权） |
| 内容红线 | 不涉及政治/色情/暴力/歧视/未成年违规 |

**3.5 流量预估**

| 维度 | 评估 |
|------|------|
| 完播率预估 | 高(>60%) / 中(40-60%) / 低(<40%) |
| 互动潜力 | 想点赞？想评论？想转发？想收藏？ |
| 搜索价值 | 有没有人主动搜这个话题？ |
| 推荐友好度 | 符合平台算法偏好吗？（前 3 秒留存、完播、互动） |

**判定**：
- 3.1-3.3 每项至少 80% 通过 → 进入 Stage 5
- 3.4 任何一条不通过 → **打回 Stage 4 修改**
- 3.5 预估完播率「低」→ 检查钩子和节奏，**打回 Stage 4 优化**
- 整体不满意 → **打回 Stage 1 重新选题**

### Stage 6: 视频生成

写 HTML + CSS + GSAP。

7 种常用布局：全屏蒙层+大字 / 左图右文 / 三/四列卡片 / 大数字冲击 / 图片+标注气泡 / 2×2 网格 / 上下分层列表。

5 种常用动画：模糊 reveal / 弹入 / 滑入 / 逐字 / 缩放穿越。

```bash
npm run check && npm run render
cp renders/*.mp4 ../rendered.mp4
```

---

### Gate 4: 最终审核

**技术质量**（GY/T 405-2024 级别）：

| 检查项 | 标准 |
|--------|------|
| 分辨率 | 1920×1080，无卡帧花屏 |
| 画面 | 无黑帧/闪帧/压缩伪影 |
| 音频 | 无杂音/BGM 不压人声/音量稳定 |
| 字幕 | 如有，时间轴同步、无错字 |
| 文件 | 能完整播放，无损坏 |

**内容质量**：

| 检查项 | 标准 |
|--------|------|
| 画面丰富 | 每个场景布局类型不同 |
| 素材真实 | 用了真实截图/图片，不是全文字 |
| 节奏合适 | 没有场景拖得太长或闪得太快 |
| CTA 清晰 | 结尾让人知道下一步做什么 |

**不通过** → 回到对应阶段修复后重新渲染。

---

### Stage 7: 音频合成

TTS 配音 + BGM 混音。BGM 音量 0.05-0.08，TTS 音量 0.9-1.0。

## 审核关卡总览

| Gate | 位置 | 审核维度 | 不通过 |
|------|------|---------|--------|
| **Gate 1** 选题 | Stage 1→2 | 钩子力·受众规模·传播力·差异化·可行性 | 重选角度 |
| **Gate 2** 素材 | Stage 3→4 | 数量≥8·多样性≥4·有高清大图·有logo | 继续收集 |
| **Gate 3** 内容 | Stage 4→5 | 丰富度·精彩度·合理性·合规·流量预估 | 回 Stage1/4 |
| **Gate 4** 终审 | Stage 6→7 | 技术质量·画面丰富·节奏·CTA | 修复bug |

## 工具 & 环境

kimi-webbridge · HyperFrames · FFmpeg · 小米 MiMo TTS · Node.js ≥ 22

## HyperFrames 规范

`class="clip"`、`data-start/duration/track-index`、GSAP `paused:true` 注册到 `window.__timelines`、`tl.set()` hard kill、`src="bgm.mp3"`、不用 `Math.random()`/`Date.now()`
