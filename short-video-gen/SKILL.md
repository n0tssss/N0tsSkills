# 短视频生成

**质量在设计阶段决定，不在渲染后修补。**

## 执行铁律

1. **Stage 2 设计先于 Stage 3 编码** — 没画完设计稿不准写 HTML
2. **每个阶段产出独立文件** — 选题/设计/脚本各自存为 .md
3. **质量自查** — 每阶段结束时 AI 自己审一遍再进入下一阶段

## 快速启动

缺参数问用户，不缺直接进入 Stage 1。

---

## 特殊类型：歌词 MV

**AI 无法听音频。LRC 必须从音乐平台官方获取。渲染后用户检查对轨。**

### 创意原则

**没有模板。没有固定规则。每句歌词 AI 自己读、自己理解、自己决定视觉方案。**

AI 拿到歌词后，逐句分析：
- 这句话在说什么？情绪是什么？（攻击/嘲讽/叙事/抒情/宣告）
- 它在整首歌里的位置？（铺垫/build-up/高潮/收尾）
- 哪些词是 punchline？哪个字需要炸开？

基于分析结果，AI 自由决定每句的：**位置、字号、颜色、动画、特效、是否逐字弹出、是否配 emoji。**

可以用的手法（不限）：
- 逐字弹入（`stagger`）、单词缩放、整句 slam-in
- 文字抖动、glow 脉冲、颜色渐变、模糊 reveal
- 全屏大字、角落小字、斜排文字、破碎散开
- emoji 旋转飞入、装饰线条随节奏出现
- 字间距拉开发光、个别关键词放大变色

**禁止的做法**：
- ❌ 所有歌词居中 + 统一字号 + 统一动画
- ❌ main+sub 两行副标题格式
- ❌ 固定位置轮换表（"第1句左边第2句右边"）
- ❌ 硬编码的"重要性1→40px"之类的映射表

### 生成方式

歌词行数多，手写 HTML 易错。用 Node 脚本生成 HTML：输入 LRC 时间戳 → AI 逐句分析歌词 → 输出每行独立 `.clip` div。

## Stage 1: 选题 & 竞品

产出 `01-选题.md`。

必须包含：
- 一句话卖点
- 竞品对比（至少找 3 个同类内容，对比播放量/角度/优缺点）
- 差异化策略

**自查**：这个选题有人看吗？和别人有什么不同？

## Stage 2: 设计（最关键）

**没有通过设计自查，不准写 HTML。**

产出 `02-设计.md`，必须包含：

### 2.1 视觉方案总览
- 配色（底色/主色/辅色/强调色/文字色）
- 字体方案（标题/正文/强调的字号字重）
- 整体氛围描述（3 个形容词）

### 2.2 段落设计（歌词MV）/ 场景设计（其他类型）

**歌词MV 必须逐段落设计**：intro / verse / chorus / bridge / outro

每个段落明确：
```
段落：Verse 1 (0:18-0:60)
底色：深红黑 #1a0000
氛围：紧张、攻击性
文字颜色：白色主文字 + 红色关键词
特效：重音时文字抖动(scale 1→1.15→1, 0.1s)、关键词放大变红
排版：双行居中，上行36px副文，下行56px正文
背景：暗红orb + 细密网格
```

**非歌词MV 必须逐场景设计**：布局 + 素材 + 文字 + 动画

### 2.3 设计自查

打印并逐项确认：

```
🎨 设计自查
━━━━━━━━━━━━━━━━━━━━━━━━
□ 配色有层次感吗？（不只一个底色到底）    [Y/N]
□ 字体有大小对比吗？（最大≥72px）         [Y/N]
□ 每个段落/场景视觉有明显区别吗？          [Y/N]
□ 有动态特效设计吗？（不只淡入淡出）       [Y/N]
□ 关键帧/moment有特殊处理吗？             [Y/N]

全Y → 进入 Stage 3 / 否则 → 继续完善设计
```

## Stage 4: 实现

写 HTML + CSS + GSAP。严格按设计稿和脚本执行，不自由发挥。

渲染前打印：

```

━━━━━━━━━━━━━━━━━━━━━━━━
□ HTML 实现了设计稿的每个段落/场景？       [Y/N]
□ 每个场景用了不同素材/背景？              [Y/N]
□ 特效按设计稿实现了？（抖动/变色/缩放等）  [Y/N]
```

## HyperFrames 核心规则

**HyperFrames 管理可见性，GSAP 只做动画。不要用 GSAP 控制元素显示/隐藏。**

1. 每个时序元素必须是 `<div class="clip" data-start="X" data-duration="Y" data-track-index="Z">`
2. `data-start/duration` 控制可见时段——HyperFrames 自动显示/隐藏，不要用 GSAP `opacity 0→1`
3. GSAP 只用于：弹入(`scale`/`y`)、抖动、变色、发光——在元素已经可见的前提下
4. **歌词 MV**：每行歌词 = 一个 `.clip` div，`data-start`=LRC 时间戳，`data-duration`=到下一句的间隔
5. **背景**：每种背景色/装饰 = 一个独立 `.clip` div，按段落时间轴切换
6. BGM = `<audio class="clip" data-start data-duration>`
7. `paused:true` GSAP timeline 注册到 `window.__timelines["main"]`
8. 不用 `Math.random()`/`Date.now()`
