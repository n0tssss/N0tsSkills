# 视频制作

用户提出想法，AI 走完整流程：**评估选题 → 确认参数 → 收集素材 → 设计脚本 → 实现渲染**。每阶段有关卡，不通过打回。

---

## 完整流程

```
用户提想法
    │
    ▼
Stage 0  选题评估   分析选题价值，和用户逐一确认
    │
    ▼
Stage 1  参数确认   用 AskUserQuestion 逐一确认视频规格
    │
    ▼
Stage 2  信息收集   调研主题，收集素材
    │
    ▼
Gate 2   素材审核   素材够吗？不够打回 Stage 2
    │
    ▼
Stage 3  脚本设计   确定结构、分镜、视觉方向
    │
    ▼
Gate 3   方案审核   方案好吗？不够打回 Stage 3
    │
    ▼
Stage 4  实现渲染   写 HTML + CSS + GSAP → MP4
    │
    ▼
Gate 4   最终审核   技术 + 内容质量
    │
    ▼
rendered.mp4
```

---

## Stage 0: 选题评估

用户提出想法后，先分析这个选题值不值得做。

**分析维度**：

- **一句话卖点**：这个视频看完能获得什么？
- **目标受众**：谁会看？什么场景看？
- **竞品情况**：同类内容多吗？我们怎么做差异化？
- **流量潜力**：话题性强吗？有人主动搜吗？适合传播吗？
- **执行可行性**：素材能收集到吗？需要多长时间？

用 `AskUserQuestion` 确认：

```
"我分析了这个选题，你觉得这个方向可以吗？"
选项：[1] 可以，继续 [2] 换个角度 [3] 我再想想
```

---

## Stage 1: 参数确认

用 `AskUserQuestion` 逐一确认。用户说"你定"就跳过该项。

**必须确认的参数**（缺什么问什么）：

```
时长：
  [1] AI 判断 [2] 15秒 [3] 30秒 [4] 60秒 [5] 自定义

视觉风格：
  [1] AI 推荐 [2] 赛博朋克 [3] 极简白 [4] 暗金 [5] 活力橙蓝
  [6] 暗夜绿 [7] 霓虹粉紫 [8] 纯黑暴力 [9] 暖橘日落 [10] 冰川蓝

视频比例：
  [1] 1920×1080 横屏 [2] 1080×1920 竖屏 [3] 其他

BGM：[1] 自动合成 [2] 不加 [3] 手动提供
TTS：[1] 不加 [2] 小米 MiMo（需提供 API Key）
```

**确认完毕后，打印汇总让用户过目**。

## Stage 2: 信息收集

**目标：把做一个视频需要的所有信息、数据、素材全部收集齐。不全不进 Stage 3。**

### 2.1 内容调研

**网站/产品/服务**（kimi-webbridge 实地访问）：
- 首页、关于页、产品/功能页、定价页、联系页——**逐个页面截图**
- 提取真实数据：用户数、年份、规模、slogan
- 找到并下载 logo（右键另存或截图裁切）
- 记录页面上所有可用文案
- 有在线 Demo 的 → 打开截图
- 有客户案例/品牌墙的 → 逐个截图

**开源项目**：
- 读 README：功能列表、技术栈、使用场景、安装方式
- 下载官方截图、Demo GIF、Logo（GitHub 仓库里找 `assets/` 或 `docs/` 目录）
- 获取真实数据：Stars、Forks、Contributors、License
- 有官网的 → kimi-webbridge 去官网截图

**游戏/影视/文化类**：
- kimi-webbridge 去 Google Images / 壁纸站搜索下载官方海报、角色图、场景截图
- Web 搜索收集：评分、获奖、经典台词、名场面描述
- 至少找到并下载 5 张不同内容的高清图

**教程/操作指南类**：
- kimi-webbridge 实际操作一遍流程，每个步骤截图
- 记录关键界面元素位置、操作顺序

**通用**：
- Web 搜索同类内容（B站/抖音/YouTube）了解竞品数据
- 搜索相关话题的讨论（知乎/公众号/Reddit）了解受众关注点

### 2.2 素材清单

收集完毕后建立清单：

```
素材/
  screenshot-home.jpg      — 首页截图
  screenshot-features.jpg  — 功能页截图
  logo.png                 — Logo
  hero-banner.jpg          — 主视觉图
  data-notes.md            — 关键数据记录
  bgm.mp3                  — 背景音乐（自动合成或用户提供）
```

### 2.3 数据核实

所有要在视频中引用的数字和事实，必须有来源：
- 用户数据 → 官网/官方公告
- Stars/下载量 → GitHub API
- 年份/历史 → 关于页面/Wikipedia
- 功能描述 → 官方文档/README

**不确定的数据宁可不用，不可瞎编。**

---

### Gate 2: 素材审核

```
□ 素材数量：≥5 个不同文件？
□ 素材多样性：不同页面/不同角度/不同内容？（不是同一张图反复用）
□ 有高清主视觉图吗？（能全屏铺满的那种）
□ 有 logo 吗？
□ 关键数据有来源吗？（不是编的）
□ 如果做这个视频需要 8 个场景，现有素材能覆盖每个场景吗？
```

**不通过 → 打回 Stage 2 继续收集，缺什么补什么。**

---

## Stage 3: 脚本设计

根据素材和参数，设计视频骨架：

- 场景划分和时长分配
- 每个场景的内容、素材、动画方向
- 配色和字体方案
- 关键帧布局

---

### Gate 3: 方案审核

```
□ 看完方案能一句话说清这个视频讲什么吗？
□ 前 3 秒能抓住人吗？
□ 每个场景有对应素材吗？
□ 场景之间有变化吗？
□ 符合用户确认的参数和方向吗？
```

---

## Stage 4: 实现渲染

```bash
npm run check && npm run render
cp renders/*.mp4 ../rendered.mp4
```

---

### Gate 4: 最终审核

```
□ 视频能正常播放？无卡帧/黑屏？
□ 内容符合设计方案？
□ 作为观众，觉得好看吗？
```

## 工具 & 环境

Node.js ≥ 22 · FFmpeg · kimi-webbridge

## HyperFrames 规则

- `class="clip"` + `data-start/duration/track-index` 管理元素时序
- GSAP `paused:true` → `window.__timelines`，只做动画不管理显示/隐藏
- orb 不同 track-index，`tl.set()` hard kill
- 不用 `Math.random()` / `Date.now()`
