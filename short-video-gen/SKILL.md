# 短视频生成

**核心规则：每道 Gate 必须实际执行并输出审核结果。跳过 Gate 直接渲染 = 违规。**

## 执行协议

每个 Gate 必须：
1. 打印审核清单和判定结果
2. 不通过 → 明确说「打回 Stage X，原因：...」
3. 通过 → 明确说「Gate X 通过」才能进入下一阶段

---

## 快速启动

缺参数问用户（时长/视觉风格/比例），不缺直接进入 Stage 1。

## Stage 1: 选题策划

产出 `01-选题报告.md`。必须包含竞品调研数据（WebSearch 搜同类内容，对比播放量/角度/差异）。

### Gate 1: 选题审核

打印并逐项检查：

```
🔍 Gate 1 选题审核
━━━━━━━━━━━━━━━━━━━━━━━━
□ 钩子够强吗？（前3秒让人停下来？）  [Y/N]
□ 受众清晰吗？（谁看？什么场景？）   [Y/N]
□ 差异化够吗？（和别人有什么不同？）  [Y/N]
□ 竞品数据有吗？（同类播放量/角度）   [Y/N]
□ 执行可行吗？（素材能收集到吗？）    [Y/N]

判定：≥4个Y → 通过 / 否则 → 打回 Stage 1
```

---

## Stage 2: 素材收集

收集所有视觉+音频素材到 `素材/`。

**针对不同视频类型的素材门槛**：

| 类型 | 硬性要求 |
|------|---------|
| 产品/网站介绍 | ≥6张不同页面截图 + logo |
| 游戏/影视 | ≥5张不同内容图（海报+角色+场景） + logo |
| 歌词MV | **音频文件已验证为正确歌曲** + 完整LRC歌词 |
| 概念/科普 | ≥3个数据可视化元素 + 相关图片 |

### Gate 2: 素材审核

```
🔍 Gate 2 素材审核
━━━━━━━━━━━━━━━━━━━━━━━━
□ 素材数量达标？（≥类型要求）          [Y/N]
□ 每个场景都有对应的素材吗？           [Y/N]
□ 音频类：已验证音频是正确内容？        [Y/N]
□ 音频类：歌词/LRC已验证完整准确？      [Y/N]
□ 图片类：至少1张高清大图（≥1280px）？  [Y/N]

判定：全Y → 通过 / 否则 → 打回 Stage 2 继续收集
```

**音频验证方法（歌词MV必须做）**：
1. kimi-webbridge 打开网易云/B站找到歌曲页面
2. 播放验证确实是这首歌
3. 对比 LRC 歌词和页面显示的歌词是否一致
4. 确认后才能通过 Gate 2

---

## Stage 3: 脚本

产出 `04-视频脚本.md`，一行一个场景。

**视觉丰富度要求（所有类型）**：
- 场景数 ≥ 6
- 不同布局类型数 ≥ 场景数 × 60%
- 相邻场景不重复布局
- 至少1个数据冲击场景、1个全屏蒙层场景

**歌词MV额外要求**：
- 按歌曲段落分区（intro/verse/chorus/bridge/outro）
- 每个段落不同的视觉配色/氛围
- 标注每段的视觉特效（抖字/放大/变色/闪烁/故障）
- 标注punchline位置及其特殊处理

### Gate 3: 内容审核

```
🔍 Gate 3 内容审核
━━━━━━━━━━━━━━━━━━━━━━━━
□ 每场景 ≤15字？                      [Y/N]
□ 前3秒有钩子？                       [Y/N]
□ 不同布局类型 ≥ 场景数×60%？          [Y/N]
□ 相邻场景不重复？                     [Y/N]
□ 每个场景指定了素材？                 [Y/N]
□ 歌词MV：每个段落不同配色？            [Y/N]
□ 歌词MV：punchline有特殊处理？         [Y/N]
□ 结尾有记忆点？                       [Y/N]

判定：全Y → 通过 / 否则 → 打回 Stage 3
```

## Stage 4: 实现

写 HTML + CSS + GSAP。

渲染前自查（必须打印）：

```
🔍 渲染前自查
━━━━━━━━━━━━━━━━━━━━━━━━
□ 每个场景用了不同的素材图/背景？      [Y/N]
□ ≥1个大数字/数据冲击场景？            [Y/N]
□ ≥1个全屏图片蒙层场景？               [Y/N]
□ 标题字号 ≥72px？                    [Y/N]
□ 没有相邻纯文字场景？                 [Y/N]
□ 歌词MV：节拍同步效果存在？            [Y/N]
□ 歌词MV：段落间视觉有明显变化？         [Y/N]

判定：全Y → 渲染 / 否则 → 回 Stage 4 修改
```

```bash
npm run check && npm run render
cp renders/*.mp4 ../rendered.mp4
```

### Gate 4: 最终审核

**视频渲染完成后必须播放检查**。打开 `rendered.mp4` 快速浏览：

```
🔍 Gate 4 最终审核
━━━━━━━━━━━━━━━━━━━━━━━━
□ 文件能正常播放？                     [Y/N]
□ 画面无卡帧/花屏/黑屏？               [Y/N]
□ 音频正常，音量合适？                 [Y/N]
□ 歌词MV：歌词和音频完全对轨？          [Y/N]
□ 视觉丰富度：画面一直有变化不单调？     [Y/N]
□ 整体感受：作为观众觉得好看吗？         [Y/N]

判定：全Y → 交付 / 否则 → 回到对应阶段修复
```

## 工具

kimi-webbridge · HyperFrames · FFmpeg · 小米 MiMo TTS

## 环境

Node.js ≥ 22 · FFmpeg · kimi-webbridge

## HyperFrames 规范

`class="clip"`、`data-start/duration/track-index`、GSAP `paused:true` → `window.__timelines`、`tl.set()` hard kill、`src="bgm.mp3"`、不用 `Math.random()`/`Date.now()`
