# 短视频生成

工业化视频创作流水线。7 个阶段，3 道审核关卡。**审核不通过打回上一阶段重做。**

## 工作目录

```
D:\视频生成\<project>/
  01-选题报告.md
  02-研究笔记.md
  03-素材清单.md
  04-视频脚本.md
  05-设计稿.md
  素材/
  index.html
  rendered.mp4
```

## 输出

默认 1920×1080。用户可指定其他比例。

## 参数

缺了用 `AskUserQuestion` 问，不缺直接开始：

- 时长 → 用户没说+配置没有就问
- 视觉风格 → 同上
- 比例/BGM/TTS → 按配置

## 流水线

### Stage 1: 选题策划 → `01-选题报告.md`

**目标**：确定这个视频「为什么值得看」。

选题报告内容：
```markdown
# 选题报告

## 一句话卖点
用一句话说清楚这个视频看完能获得什么

## 目标受众
谁会看？在什么场景下看？（通勤刷手机？电脑前摸鱼？）

## 核心钩子
前 3 秒用什么抓住注意力？
- 反常识？"你以为 XXX，其实不是"
- 大数字？"28 年，10 万+ 品类"
- 痛点？"电子元器件采购有多痛苦？"
- 好奇？"这个网站 1997 年就上线了"

## 竞品分析
同类题材的视频怎么做？我们有什么不同？

## 预估吸引力
- 受众规模：大/中/小众
- 传播力：强/中/弱（话题性、实用价值、情绪共鸣）
- 差异化：和同类视频的区分度

## 风险点
- 素材可能不足？
- 话题太专业观众看不懂？
- 信息密度太高或太低？
```

**审核关卡 Gate 1**：
- 选题报告写完，AI 自己审一遍：
  - [ ] 前 3 秒钩子够不够强？不够 → 重写钩子
  - [ ] 这个选题目标受众清晰吗？不清晰 → 重新定位
  - [ ] 差异化够吗？和别人做的一样 → 换角度

---

### Stage 2: 内容研究 → `02-研究笔记.md`

**目标**：深挖主题，收集所有可能需要的内容素材。

**网站/产品**：
- kimi-webbridge 逐个页面截图：首页、关于、产品、案例、联系
- 记录页面上所有可引用的数据和文案
- 找到并下载 logo、产品图、客户案例图

**开源项目**：
- 读 README 提取功能列表、技术栈、使用场景
- 下载官方截图、Demo GIF、Logo
- 记录 Stars/下载量/贡献者数

**其他题材**：
- Web 搜索收集背景资料
- Google Images 搜索下载相关图片

研究笔记内容：
```markdown
# 研究笔记

## 核心信息
- 是什么？解决什么问题？
- 关键数据（数字、年份、规模）
- 独特卖点（3-5 个）
- 目标用户的真实痛点

## 素材清单
列出现在手上有哪些可用的视觉素材
```

---

### Stage 3: 素材收集 → `03-素材清单.md`

**目标**：凑够做视频需要的所有视觉素材。

**硬性门槛**（不达标不进 Stage 4）：
- [ ] 素材目录 ≥8 个文件
- [ ] ≥4 张不同内容的图（不同页面/不同角度/不同产品）
- [ ] 有 logo
- [ ] 至少有 1 张能全屏铺满的高清图（≥1280px 宽）

**审核关卡 Gate 2**：
- 逐个检查素材：
  - [ ] 每张图能用在哪个场景？说不出来 → 这张图没用，删掉重找
  - [ ] 4 张不同的图够支撑 6-8 个场景吗？不够 → 回去搜更多
  - [ ] 有没有能制造视觉冲击的图？（大数据/全景/对比/人物）没有 → 专门去找

---

### Stage 4: 脚本创作 → `04-视频脚本.md`

**目标**：写出有节奏、有钩子、有记忆点的分镜脚本。

脚本格式：
```markdown
# 视频脚本

| # | 时间 | 布局 | 素材 | 画面描述 | 画面文字 | 台词 | 动画 |
|---|------|------|------|---------|---------|------|------|
| 1 | 0-4s | 全屏蒙层 | hero大图 | 暗色蒙层+大字居中 | "XXX" | "完整口播" | 模糊reveal |
```

脚本原则：
- 前 3 秒必须有钩子（反常识/大数字/痛点/好奇）
- 每 5-8 秒换场景
- 相邻场景用不同布局（全屏→卡片→分栏→大数字→全屏...）
- 每屏文字 ≤15 字
- 结尾必须让人记住点什么（一句话/一个数字/一个地址）
- 每行标注使用素材里的哪张图

**审核关卡 Gate 3**：
- [ ] 看完脚本能说出一句话总结吗？说不出来 → 主线不清晰，重写
- [ ] 前 3 秒能让人停下来吗？不能 → 钩子太弱，重写前 3 秒
- [ ] 每个场景都指定了素材图吗？没有 → 补上
- [ ] 场景类型 ≥ 总场景数的 60% 吗？（8 场景至少 5 种不同布局）
- [ ] 有数据/数字冲击场景吗？没有 → 加一个
- [ ] 结尾有记忆点吗？没有 → 加

---

### Stage 5: 视觉设计 → `05-设计稿.md`

确定配色方案、字体层级、关键帧布局、动画方案。

10 种配色可选，根据题材自由发挥：

| 风格 | 底色 | 主色 | 适合 |
|------|------|------|------|
| 极简白 | `#fafafa` | 蓝紫渐变 | 商务/SaaS/教程 |
| 纯黑暴力 | `#000` | 纯白+红 | 游戏/影视/冲击 |
| 暗金 | `#0c0c0c` | 金色渐变 | 高端/品牌/奢侈品 |
| 赛博朋克 | `#0a0a12` | 紫+青绿 | 科技/AI/Web3 |
| 暗夜绿 | `#0a0f0a` | 绿青渐变 | 安全/终端/黑客 |
| 冰川蓝 | `#f0f5fa` | 深蓝渐变 | 医疗/金融/专业 |
| 暖橘日落 | `#1a100a` | 橙粉渐变 | 生活/温暖/故事 |
| 红黑警戒 | `#0a0000` | 红色渐变 | 促销/紧迫/警告 |
| 霓虹粉紫 | `#0f0a14` | 粉紫渐变 | 潮流/年轻/音乐 |
| 活力橙蓝 | `#0f0f1a` | 橙蓝渐变 | 运动/活力/创新 |

### Stage 6: 视频生成

写 HTML + CSS + GSAP。

常用布局模式：
- **全屏图片+蒙层+大字** — 开场/章节分隔/结尾
- **左图右文** — 截图+特性说明
- **三/四列卡片** — 功能罗列/数据展示
- **大数字冲击** — 年份/规模/数据
- **图片+标注气泡** — 截图细节说明
- **2×2 网格卡片** — 名场面/案例展示
- **上下分层列表** — 步骤教程

常用动画：
- 模糊 reveal：`blur(20px)→0`
- 弹入：`scale(0.8)→1, back.out`
- 滑入：`x:±60→0` 或 `y:40→0`
- 逐字：stagger per-character

过渡：淡入淡出 0.25s out + 0.3s in；缩放穿越、滑动切换可选。

渲染前自查：
- [ ] 每个场景用不同的素材图
- [ ] ≥1 个大数字场景
- [ ] ≥1 个全屏蒙层场景
- [ ] 场景布局多样性 ≥50%
- [ ] 标题 ≥72px
- [ ] 没有相邻纯文字场景

```bash
npm run check && npm run render
cp renders/*.mp4 ../rendered.mp4
```

---

### Stage 7: 音频合成

如果配置了 TTS：为脚本每句台词生成配音 WAV，在 HTML 中为每句台词添加 `<audio>` 标签。BGM 音量压低到 0.05-0.08。

## 审核关卡总览

| 关卡 | 位置 | 检查内容 | 不通过 |
|------|------|---------|--------|
| Gate 1 | Stage 1 后 | 钩子够强？受众清晰？有差异化？ | 回到 Stage 1 |
| Gate 2 | Stage 3 后 | 素材 ≥8 文件？≥4 不同图？有 logo？有高清大图？ | 回到 Stage 2/3 |
| Gate 3 | Stage 4 后 | 主线清晰？前 3 秒够强？素材都用上了？布局多样？ | 回到 Stage 4 |

## 工具

kimi-webbridge（浏览器调研+搜图）· HyperFrames（HTML→MP4）· FFmpeg（BGM）· 小米 MiMo TTS（配音）

## 环境

Node.js ≥ 22 · FFmpeg · kimi-webbridge

## HyperFrames 规范

`class="clip"`、`data-start/duration/track-index`、GSAP `paused:true` 注册到 `window.__timelines`、`tl.set()` hard kill、`src="bgm.mp3"`、不用 `Math.random()`/`Date.now()`
